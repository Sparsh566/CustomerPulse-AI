SLIDE 6 — CORE AI & NLP FEATURES
================================================================================

Heading: Core AI / NLP Intelligence Engine

AI / NLP Features:

1. Auto-Categorization
   Product type (loan, card, UPI, insurance) + complaint type (fraud, delay,
   mis-selling) classification. Confidence score shown to agent with manual
   override option.

2. Duplicate & Cluster Detection
   Semantic similarity cross-reference on incoming complaints. Prevents
   duplicate effort. Surfaces systemic product failures faster.

3. Root Cause Identification
   Finds repeating failure patterns across complaint clusters.
   Example: "UPI timeout errors from a specific bank partner."
   Actionable signals for operations teams.

4. Sentiment & Severity Scoring
   Every message scored sentiment (positive / neutral / negative / critical)
   with confidence percentage. Severity score 1–10 for priority.
   Sentiment drift tracked — if mood worsens across messages, auto-flag.

5. Predictive SLA Breach Detection
   Breach probability calculated per complaint based on complexity, agent
   workload, priority, and time remaining. Auto-reassignment if risk > 80%.

6. Trend & Spike Detection
   Complaint volume and sentiment spikes tracked over time.
   Manager flagged before it becomes a PR or regulatory issue.
   Proposed: Predictive spike forecasting using 30-day rolling data.

Gen-AI Capabilities:

1. AI Draft Response Generation
   Agent selects tone: formal / empathetic / apologetic / neutral.
   Gen-AI writes complete customer-facing response ready for review.
   Response drafting time: 8 minutes → under 60 seconds.
   Proposed: Multi-language responses — English, Hindi, Tamil.

2. Resolution Templates & Next-Best Actions
   AI suggests most relevant resolution and action based on category +
   severity + past cases. Examples: "Initiate refund" / "Schedule callback"
   / "Escalate to Level 2".

3. Similar Case Suggester
   Agent sees: "5 similar cases resolved in 2 hours — here's how."
   Semantic search across all resolved complaints.

4. Regulatory Report Generation
   One-click RBI Annexure-style reports. Auto-populated data.
   Exportable as CSV and PDF.

5. AI Insights & Trend Narrative
   Executive summary: what spiked, why, which products, which root causes,
   what actions are recommended — all in plain English.

6. AI Operations Assistant (Streaming Chat)
   Natural language queries on full dataset. No SQL needed.
   Example queries:
   "Which agent has most SLA breaches this month?"
   "Show all fraud complaints from last week."
   "Which customers have complained 3+ times this quarter?"
   "Show all CRITICAL urgency cases still open."

Heading: What Makes CustomerPulse AI Different

────────────────────────────────────────────────────────────────────────────────
FEATURE 1: Customer Emotion Timeline [NEW]
────────────────────────────────────────────────────────────────────────────────

What it is:
Tracks the full emotional journey of a complaint — not just a static sentiment
score. Every message gets an emotion label. Every agent reply is checked to see
if it helped or hurt the customer's mood.

Emotion Labels:
Calm → Anxious → Frustrated → Angry → Hopeful → Relieved → Disappointed

What it tracks:
- Sentiment score per message (positive / neutral / negative)
- Emotion label per message
- Trigger detection — did the agent's reply worsen the mood?
- Recovery score — did a good reply improve the mood?
- Final emotion state at closure — "resolved but unhappy" flagged as churn risk

Example journey:
Message 1  — Customer reports issue — Calm
Agent Reply 1 — Copy-paste generic response sent — Frustration spike
Message 2  — "I already told you this" — Angry
Agent Reply 2 — Agent acknowledged wait + escalated — Slight relief
Message 3  — Customer hopeful but tired
Resolution — Satisfied but not delighted (score: 68/100)

4 AI Agents running on Emotion Timeline:

Empathy Coach Agent
→ If 2 consecutive replies worsen mood, whispers to agent:
  "Acknowledge the wait before explaining."

De-escalation Trigger Agent
→ When emotion crosses anger threshold, auto-loops in senior agent
  without waiting for human decision.

Resolution Satisfaction Predictor
→ "OK fine" closures flagged as churn risk.
→ Predicts if customer will complain again in 30 days.

Root Cause Emotion Agent
→ Compares timelines across all customers.
→ Finds which specific process steps consistently trigger anger.
→ Example: "Verification call step frustrates 73% of customers."

Tagline: "Detects when YOUR reply made the customer angrier — and tells you."

────────────────────────────────────────────────────────────────────────────────
FEATURE 2: Financial Urgency Score Engine [NEW]
────────────────────────────────────────────────────────────────────────────────

What it is:
A 0–100 score calculated for every complaint based on the real financial,
medical, or legal harm caused by delay. Genuine emergencies get genuine priority.
Not just a "High Priority" tag — an actual measured urgency level.

Score is calculated from:
- Harm type (Medical > Legal > Financial > Education > Travel > Business)
- Verifiability of the claim (specific details vs vague statement)
- Time to harm (< 2 hrs / < 24 hrs / < 72 hrs — weighted accordingly)

8 Urgency Categories (Genuine Cases Only):

CRITICAL — Medical Emergency
ICU deposit stuck, dialysis payment failed, hospital admission blocked,
ambulance payment declined, overseas medical emergency with blocked card.

CRITICAL — Legal / Court Deadline
Court fee last date, bail bond window closing, property registration deadline,
legal notice demand draft failed.

HIGH — Financial Penalty
EMI bounce due to bank error, credit card late fee accumulating per day,
tax filing deadline, GST late fee starting.

HIGH — Education Critical
College seat cancellation imminent, scholarship amount not received,
exam form fee failed, overseas university wire transfer stuck.

MEDIUM — Travel Emergency
Flight in 3-4 hours, ticket payment failed, overseas stranded with blocked
card, visa fee deadline.

MEDIUM — Business / Livelihood
Salary day tomorrow, account frozen — employees affected. Vendor penalty
clause active. GST filing deadline.

STANDARD — Government Deadlines
PM Kisan / scholarship / subsidy last date. ITR filing deadline.

STANDARD — Utility / Basic Need
Electricity bill last date (disconnection risk). Gas cylinder payment failed.

Genuineness Filter (prevents abuse):
- AI cross-checks deadline against actual account due dates
- Specific details (hospital name, court number, flight number) increase score
- Repeat CRITICAL claimers with no matching account activity: score reduced + flagged
- MCC code verification for hospital/court payment failures

Score → Action:
80–100 CRITICAL → Outbound call within 15 mins, senior agent mandatory
60–79  HIGH     → Top of queue, same-day resolution
40–59  MEDIUM   → Flagged, same-day queue
0–39   STANDARD → Regular SLA applies

Tagline: "A hospital payment stuck at 2am gets treated like the emergency it is."

────────────────────────────────────────────────────────────────────────────────
FEATURE 3: Customer 360 Profile & Complaint History Database [NEW]
────────────────────────────────────────────────────────────────────────────────

What it is:
Every customer has a dedicated profile with their complete complaint history —
visible to every agent before they respond. No more blind handling.

What the profile contains:
- Customer identity: name, contact, account type, segment, join date
- Total complaints ever raised
- Complaint categories and channels used
- Resolution times and SLA compliance per complaint
- CSAT score per complaint
- Escalation count
- Emotion state at each closure
- Urgency tier history
- Repeat complaint patterns (e.g., "UPI failure — 4 times in 60 days")
- Overall churn risk score (AI-calculated)
- Agent note: what to be careful about next time

Example profile:
Customer: Rajesh Kumar | Savings Account | Since 2019
Total Complaints: 7 | Escalated: 2 | Avg CSAT: 3.2/5
Repeat Category: UPI Failure (4 times in 3 months)
Churn Risk: HIGH
Last Emotion at Closure: Frustrated
Agent Note: "Acknowledge history before asking for documents — been through this before."

Pattern detection:
"This customer has complained about UPI 4 times in 60 days." → System flags
product-level failure, not just customer-level issue.

Proposed addition:
Automated VIP flag for high-value customers with repeated failures.

Tagline: "No more blind handling — agents know exactly who they're talking to."

────────────────────────────────────────────────────────────────────────────────
BONUS FEATURE: Agent Whisper Assistant [NEW]
────────────────────────────────────────────────────────────────────────────────

What it is:
Real-time AI suggestions shown only to the agent during a live complaint.
What to say next. When to escalate. How to de-escalate.

How it works:
- Triggered by live complaint text + emotion signals + urgency score
- Suggestions appear in a side panel, invisible to customer
- Agent focuses on the customer — AI handles the strategy

Example suggestions:
"Acknowledge the medical stress first before asking for documents."
"Offer refund — 3 similar cases resolved this way in < 2 hours."
"Escalate now — emotion threshold crossed + CRITICAL urgency."

Combined power:
If customer has CRITICAL urgency + rapidly worsening emotion timeline,
system raises "Intervention Required" flag + specific agent instruction.
This directly solves the real banking failure — document-first approach
when customer is already in panic.
